# i18n（多語系）設計 — koikatsu-dedupe

日期：2026-06-26
狀態：已核准設計，待寫實作計畫

## 1. 目標

讓 GUI 支援 7 種 KK 社群常用語言，使用者可在右上角即時切換，選擇記住。
維持專案核心精神：**單一靜態前端、無建置步驟、零執行期依賴**。

語言（顯示順序＝切換器順序，名稱用母語名）：

| code | 母語名 | 備註 |
|---|---|---|
| `zh-Hant` | 繁體中文 | 現有原始字串、**參考語言（fallback 基準）** |
| `zh-Hans` | 简体中文 | |
| `en` | English | **偵測失敗時的最終 fallback** |
| `ja` | 日本語 | |
| `ko` | 한국어 | |
| `ru` | Русский | 需複數規則（3 形） |
| `es` | Español | 需複數規則（單/複） |

## 2. 首次語言決定

```
lang = localStorage['dedupe.lang']  ?? detectLang(navigator.language)
```

`detectLang(tag)` 映射規則（不分大小寫，取主語言子標籤）：

| 來源 tag | → code |
|---|---|
| `zh-TW` / `zh-HK` / `zh-MO` / `zh-Hant*` | `zh-Hant` |
| 其餘 `zh*`（`zh-CN` / `zh-SG` / `zh-Hans*` / 裸 `zh`） | `zh-Hans` |
| `en*` | `en` |
| `ja*` | `ja` |
| `ko*` | `ko` |
| `ru*` | `ru` |
| `es*` | `es` |
| 其他一律 | `en`（最終 fallback） |

## 3. 架構

純 vanilla、零依賴、不破壞 no-build。**不引入任何函式庫或打包器。**

新增單一檔案 `dist/i18n.js`，以 `<script src="i18n.js"></script>` 在主 `<script>` 之前載入（file:// 與 Tauri 皆可運作，相對路徑）。匯出（掛在 `globalThis`）：

- `I18N` — 7 語言字典物件，以穩定 key 索引（見附錄 A）。
- `LANGS` — `[{code, native}]` 有序清單，供切換器使用。
- `detectLang(tag)` — 第 2 節映射，回傳 code。
- `setLang(code)` / `getLang()` — 設定/讀取目前語言（模組內狀態）。
- `t(key, params)` — 查 `I18N[lang][key]`，缺則 fallback `zh-Hant`，再缺回傳 key 本身（不崩）。`params` 內以 `{name}` 形式內插；數字用 `toLocaleString(lang)`。
- 複數：字典值可為純字串，或物件 `{one?, few?, many?, other}`。`t()` 偵測到物件時，依 `pluralCategory(lang, params.n)` 取對應形，再內插。
  - `zh-Hant/zh-Hans/ja/ko` → 永遠 `other`
  - `en/es` → `n===1 ? 'one' : 'other'`
  - `ru` → 標準俄語規則（`one` / `few` / `many`）

### 3.1 `index.html` 改動

**靜態 HTML** — 每個使用者可見文字節點加標記，由 `applyStatic()` 走訪 `[data-i18n],[data-i18n-html],[data-i18n-placeholder],[data-i18n-title]` 填入：

- `data-i18n="key"` → 設 `textContent`
- `data-i18n-html="key"` → 設 `innerHTML`（用於含 `<code>`/`<b>`/`<ul>` 的「How it works」段落、review 提示、summary 警告；內容為受信任的靜態譯文）
- `data-i18n-placeholder="key"` → 設 `placeholder`
- `data-i18n-title="key"` → 設 `title`

**動態 JS 字串** — 原地字面值改為 `t('key', {...})`。涵蓋：toast、`第 N 組 / 共 M 組`、`已選 N 張`、逐行比對 tally、sync 完成/錯誤訊息、`PHASE` 對照、`availLabel`/`refreshAvail`、`refreshRootStatus`、`renderGroup` 卡片按鈕、`renderDetail`。

**導覽圈 `ANN`** — 改為在 `drawTour()` 內以 `t('tour.*')` 取文字（`drawTour` 本就在切換語言時被呼叫，故自動重繪在地化）。

**語言切換器** — header 右上新增 `<select id="langSel">`，選項取自 `LANGS`（母語名）。`onchange`：
1. `setLang(code)`；`localStorage['dedupe.lang']=code`
2. `applyStatic()`
3. 重繪目前動態視圖（依 `CURVIEW`：review→`showGroup()`、其餘 re-apply 已足夠；availTxt/rootStatus 重算）
4. `document.documentElement.lang = code`
5. 更新視窗標題（見 3.3）
6. `drawTour()`

**初始化** — 載入時決定 lang（第 2 節）→ `setLang` → `applyStatic` → 設 `<html lang>` → 設標題 → 既有流程照舊。切換器目前值同步。

### 3.2 後端錯誤碼在地化

`src-tauri/src/lib.rs` 兩個手寫中文錯誤改回傳**穩定碼**（不再是中文）：

- `lib.rs:74` `"來源路徑無檔名"` → `"src_no_name"`
- `lib.rs:77` `format!("遊戲 chara 路徑不存在或不是資料夾：{}", …)` → `format!("game_path_invalid|{}", dest.display())`

前端 `copyCard` 的 `catch` 端解析：若 `e` 為已知碼前綴 → 用 `t('err.src_no_name')` / `t('err.game_path_invalid',{path})`；否則原樣顯示（`std` 錯誤多為英文，可接受）。

其餘 Rust 錯誤（`e.to_string()` 來自 std/rusqlite）維持原樣 — 技術性、偶發、英文，超出本次範圍。

### 3.3 視窗標題

Tauri 原生標題列來自 `tauri.conf.json`，與 `document.title` 不同。

- 執行期：`getCurrentWindow().setTitle(t('app.title'))`（透過 `window.__TAURI__`；file:// 下 try/catch 優雅略過）。同時設 `document.title`。
- `src-tauri/capabilities/default.json` 的 `permissions` 加 `"core:window:allow-set-title"`。
- `tauri.conf.json` 的 `app.windows[0].title` 改為中性 fallback `"Koikatsu Dedupe"`（執行期立即被在地化標題覆蓋；僅在極早期/權限缺失時短暫顯示）。

## 4. 翻譯策略

1. 先定**術語表**（附錄 B）確保 7 語言用詞一致。
2. 集中翻譯全部 key（單一作者，保證術語一致）。
3. **分語言母語級 QA**：每語言一個 reviewer 平行校對（自然度、術語、複數、版面長度）。
4. 譯文為 AI 產出，交付時標註「建議母語者複審」。

## 5. 不在範圍

- 純 file:// mock 字串：假卡名 `NAMES`（さくら…）、`makeGroups` 的 `dup#`、`syncSteps` 假進度 log、`availLabel` 的 mock 數字。真實 app 不顯示這些。（共用 key 的 chrome 文字仍在地化，純 mock 數據不動。）
- `core.rs` 內的 SQL/技術字串、`std` 錯誤訊息。
- 數字以外的進階地區化（日期格式等）— 本 app 無日期顯示。
- RTL — 7 語言皆 LTR，無需處理。

## 6. 測試與驗收

### 自動
- 新增 `scripts/check-i18n.mjs`（純 Node、零依賴）：載入 `i18n.js`，斷言
  1. 每個語言的 key 集合與參考語言 `zh-Hant` **完全相同**（無多、無缺）；
  2. 無空字串值；
  3. 凡 `zh-Hant` 標為複數物件的 key，`ru`/`en`/`es` 必須提供對應複數形。
  失敗則 `process.exit(1)`。文件化執行指令：`node scripts/check-i18n.mjs`。
- `cd src-tauri && cargo test` 仍全綠（僅錯誤碼字串變動，邏輯不變）。

### 手動冒煙
- 逐一切換 7 種語言，走完 sync→setup→review→summary 四步：無漏字（無殘留中文、無 raw key）、無版面爆框、導覽圈與「How it works」皆在地化、切換即時生效且重開記住。
- 俄語複數：`組/張/列` 在 n=1/2/5/21 顯示正確形。

### 成功標準（量化）
1. 7 種語言，每種 100% key 覆蓋（parity 測試通過）。
2. 0 個使用者可見字串在切換後仍顯示參考語言以外語言的殘留（手動冒煙確認 4 步全覆蓋）。
3. `cargo test` 0 失敗。
4. 新增執行期依賴數 = 0；新增建置步驟數 = 0。
5. `index.html` 仍可直接以 file:// 開啟並運作（mock 模式不崩）。
